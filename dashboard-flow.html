4 ESP32: Procesamiento y Visualización

Firmware Arduino en C++ — Dual core: Core 0 = HTTP fetch, Core 1 = Touch + Display
# Arquitectura dual-core:

Core 0 — Hilo de red (fetchTask)
  ├─ Cada 2s hace HTTP GET al servidor
  ├─ Parsea JSON con ArduinoJson
  ├─ Guarda datos en variables globales (con mutex)
  └─ Señala fetchDone = true

Core 1 — Loop principal (50Hz)
  ├─ Lee touch XPT2046 cada 20ms
  ├─ Si hay touch → cambia vista instantáneamente
  ├─ Si fetchDone → redibuja con datos nuevos
  └─ Dibuja en ILI9488 por SPI a 40MHz

# Pines SPI (compartido display + touch):
  CLK  = GPIO14    CS Display = GPIO15
  MOSI = GPIO13    CS Touch   = GPIO33
  MISO = GPIO12    DC         = GPIO2
  Backlight = GPIO27    Touch IRQ = GPIO36

# 5 Vistas (touch para ciclar):
  0 = Dashboard completo
  1 = CPU detallado + gráfica grande
  2 = RAM/Disco detallado + gráfica grande
  3 = CPU Cores pantalla completa
  4 = Procesos pantalla completa

Flujo paso a paso (cada 2 segundos)

1 psutil lee el sistema
Python llama a las APIs de Windows para obtener CPU, RAM, disco, red y procesos actuales. Todo en una sola función get_system_stats()
→
dict Python
2 FastAPI serializa a JSON
El endpoint /api/stats convierte el diccionario Python a JSON y lo envía como respuesta HTTP. ~2KB de datos.
→
HTTP sobre WiFi
3 ESP32 Core 0 recibe JSON
El hilo fetchTask en Core 0 hace HTTP GET, recibe el JSON y lo parsea con ArduinoJson. Guarda los valores en variables globales protegidas con mutex.
→
variables globales
4 ESP32 Core 1 dibuja en pantalla
El loop principal detecta fetchDone y redibuja la vista actual. Usa TFT_eSPI para dibujar gauges, gráficas, barras y texto sobre SPI a 40MHz al ILI9488.
→
SPI 40MHz
5 Pantalla TFT muestra datos
ILI9488 480x320 16-bit color muestra las gráficas. El touch XPT2046 permite cambiar entre 5 vistas tocando la pantalla.
